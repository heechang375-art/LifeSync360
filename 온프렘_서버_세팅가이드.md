# 온프렘 서버 세팅 가이드
> 환경: VirtualBox + Ubuntu 24.04 Server × 3대  
> 역할: MySQL VM / Tokenization VM / Private API VM

---

## 서버 구성 목표

| 항목 | MySQL VM | Tokenization VM | Private API VM |
|------|----------|-----------------|----------------|
| 호스트명 | `ls-db` | `ls-token` | `ls-api` |
| 고정 IP (Host-Only) | `192.168.56.11` | `192.168.56.12` | `192.168.56.13` |
| 주요 소프트웨어 | MySQL 8.0 | Python + uvicorn | Python + uvicorn + nginx |
| 포트 | 3306 | 8000 | 80 / 8000 |

---

## Step 1 | VirtualBox 네트워크 설정

> 3대 모두 동일하게 적용. VM이 꺼진 상태에서 설정.

### 호스트 전용 어댑터(Host-Only Network) 생성

VirtualBox 메인 화면 → **파일 → 호스트 네트워크 관리자**

- **만들기** 클릭
- IPv4 주소: `192.168.56.1`
- 서브넷 마스크: `255.255.255.0`
- DHCP 서버: **비활성화** (고정 IP 직접 설정할 예정)
- **적용**

### 각 VM 네트워크 어댑터 설정

VM 선택 → **설정 → 네트워크**

**어댑터 1**
- 연결 방식: `NAT`
- 용도: 인터넷 (패키지 설치, AWS 연동)

**어댑터 2**
- 연결 방식: `호스트 전용 어댑터`
- 이름: 위에서 만든 어댑터 선택
- 용도: VM 간 통신

## Step 2 | VM 공통 초기화 (3대 모두 동일)

> 각 VM 부팅 후 아래 명령어 순서대로 실행.

### 패키지 업데이트

```bash
sudo apt update && sudo apt upgrade -y
```

### 필수 패키지 설치

```bash
sudo apt install -y python3 python3-pip curl net-tools ufw openssh-server
```

### Ansible 전용 계정 생성

```bash
# ansible 계정 생성
sudo useradd -m -s /bin/bash ansible

# sudo 권한 부여
sudo usermod -aG sudo ansible

# 비밀번호 없이 sudo 실행 (Ansible 자동화 필수)
echo "ansible ALL=(ALL) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/ansible
sudo chmod 440 /etc/sudoers.d/ansible

# SSH 키 디렉토리 준비 (나중에 Ansible Control Node 공개키 등록)
sudo mkdir -p /home/ansible/.ssh
sudo chmod 700 /home/ansible/.ssh
sudo chown -R ansible:ansible /home/ansible/.ssh
```

### SSH 서비스 활성화

```bash
sudo systemctl enable ssh
sudo systemctl start ssh
```

## Step 3 | 고정 IP 설정 (VM마다 IP 다름)

### 네트워크 인터페이스 이름 확인

```bash
ip a
# enp0s3 → NAT 어댑터 (DHCP)
# enp0s8 → Host-Only 어댑터 (고정 IP 설정 대상)
```

> 이름이 다를 수 있으니 반드시 `ip a`로 확인 후 적용.

### netplan 설정 파일 수정

```bash
sudo nano /etc/netplan/50-cloud-init.yaml
```

**MySQL VM (192.168.56.11)**
```yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: true
    enp0s8:
      addresses:
        - 192.168.56.11/24
```

**Tokenization VM (192.168.56.12)**
```yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: true
    enp0s8:
      addresses:
        - 192.168.56.12/24
```

**Private API VM (192.168.56.13)**
```yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: true
    enp0s8:
      addresses:
        - 192.168.56.13/24
```

### 설정 적용 및 확인

```bash
sudo netplan apply

# 적용 확인
ip a show enp0s8
# inet 192.168.56.xx/24 가 보이면 성공
```

## Step 5 | VM 간 통신 확인

> 3대 설정이 완료되면 서로 통신이 되는지 반드시 확인.

```bash
# MySQL VM에서
ping -c 3 192.168.56.12   # ls-token

# Tokenization VM에서

ping -c 3 192.168.56.12   # ls-token
```

> 모두 응답이 오면 네트워크 설정 완료.

## Step 6 | MySQL VM 추가 설정

### MySQL 8.0 설치

```bash
sudo apt install -y mysql-server
sudo systemctl enable mysql
sudo systemctl start mysql

# 설치 확인
sudo systemctl status mysql
```

### root 비밀번호 설정

```bash
sudo mysql
```

```sql
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '변경할비밀번호';
FLUSH PRIVILEGES;
EXIT;
```

### 외부 접속 허용 (Tokenization VM, Private API VM에서 접근용)

```bash
# bind-address 변경 (기본값 127.0.0.1 → 0.0.0.0)
sudo sed -i "s/^bind-address.*/bind-address = 0.0.0.0/" \
  /etc/mysql/mysql.conf.d/mysqld.cnf

sudo systemctl restart mysql
```

### 방화벽 설정

```bash
# MySQL 포트: 같은 서브넷에서만 허용
sudo ufw allow from 192.168.56.0/24 to any port 3306

# SSH는 전체 허용
sudo ufw allow OpenSSH

# 방화벽 활성화
sudo ufw enable

# 확인
sudo ufw status
```

## Step 7 | Tokenization VM 기본 설정

```bash
# Python 패키지 미리 설치 (Ansible이 설치하겠지만 미리 준비)
```

## Step 8 | Private API VM 기본 설정

```bash
# nginx 미리 설치
sudo apt install -y nginx python3-pip

sudo systemctl enable nginx
sudo systemctl start nginx

# 방화벽
sudo ufw allow from 192.168.56.0/24 to any port 8000
sudo ufw allow 'Nginx Full'
sudo ufw allow OpenSSH
sudo ufw enable
```

## Step 10 | Ansible 사용 가능할 때 진행

> VPN 연결 + Ansible Control Node EC2 준비 완료 후 진행.

### Control Node에서 SSH 키 생성 및 배포

```bash
# Ansible Control Node (EC2)에서
ssh-keygen -t rsa -b 4096 -f ~/.ssh/lifesync360-onprem -N ""

# 각 VM에 공개키 등록
ssh-copy-id -i ~/.ssh/lifesync360-onprem.pub ansible@192.168.56.11
ssh-copy-id -i ~/.ssh/lifesync360-onprem.pub ansible@192.168.56.12
ssh-copy-id -i ~/.ssh/lifesync360-onprem.pub ansible@192.168.56.13

# 접속 테스트
ssh -i ~/.ssh/lifesync360-onprem ansible@192.168.56.11
```

### Ansible 연결 테스트

```bash
cd /opt/ansible
ansible all -i inventory/hosts.yml -m ping
```

> 3대 모두 `pong` 응답 오면 준비 완료.

### Playbook 실행

```bash
# 전체 배포
ansible-playbook site.yml -i inventory/hosts.yml

# 특정 서버만 배포
ansible-playbook site.yml -i inventory/hosts.yml --limit mysql
ansible-playbook site.yml -i inventory/hosts.yml --limit tokenization
ansible-playbook site.yml -i inventory/hosts.yml --limit private_api
```

## 진행 체크리스트

```
[지금 바로]
  □ VirtualBox 호스트 전용 어댑터 생성
  □ MySQL VM    — 어댑터 2개 설정, OS 초기화, 고정 IP, 호스트명, MySQL 설치
  □ Token VM    — 어댑터 2개 설정, OS 초기화, 고정 IP, 호스트명
  □ API VM      — 어댑터 2개 설정, OS 초기화, 고정 IP, 호스트명, nginx 설치
  □ VM 간 ping 테스트 통과

[IaC/VPN 완료 후]
  □ Ansible Control Node에서 SSH 키 배포
  □ ansible all -m ping 테스트
  □ ansible-playbook site.yml 실행
  □ MySQL DB 스키마 적재 확인
  □ 각 서비스 health check 확인
     - http://192.168.56.12:8000/health  (tokenization)
     - http://192.168.56.13/health       (private api)
```
